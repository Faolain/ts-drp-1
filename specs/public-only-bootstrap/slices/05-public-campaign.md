# Slice 05: Public Measurement Campaign

## Contract unlocked

A measured verdict across target browsers and two materially distinct network
conditions, without introducing owned discovery or relay infrastructure.

## API seam

Add a separate manual protected workflow for the public-only runner. Do not
weaken or retrofit the Phase 09 registry-shaped campaign. Reuse the public-only
contract, request ledger, sanitizer, and exact acknowledgement.

One condition may be the local operator network and the other an ephemeral
GitHub-hosted runner; the runner is measurement egress, not bootstrap or relay
infrastructure. Record that distinction.

## Runnable surface and verification

- Start with one fresh identity per target browser/network as a smoke gate.
- Only after the smoke gate is valid, run the pre-registered repeated matrix.
- Record provider visibility, browser-dialable address rate, HOP ratio,
  reservation status/limits/expiry, cold-start time, mesh/sync/direct times,
  relay/direct bytes, cleanup, package/browser versions, and sanitized coarse
  diversity.
- Freeze thresholds and request caps before traffic; never change them after
  seeing observations.
- Run schema/coverage tests, typecheck, lint, Grok, and Kimi review. Preserve
  raw artifacts only in protected short-lived storage; commit aggregate
  sanitized evidence and the honest verdict.

## Feedback that changes this slice

Authorization for public traffic, the exact second network condition, and any
operator terms required by the public utility providers.
